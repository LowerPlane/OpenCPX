# Getting Involved with OpenCPX

Welcome to the OpenCPX community! There are many ways to get involved, regardless of your background or experience level.

## Ways to Contribute

### For Everyone

#### Use OpenCPX
The best way to support OpenCPX is to use it! Implement the standard in your applications and provide feedback.

#### Report Issues
Found a bug or have a suggestion? Open an issue on GitHub. Good bug reports help us improve the project.

#### Answer Questions
Help others in GitHub Discussions. Sharing your knowledge helps build a stronger community.

#### Spread the Word
- Star the repository on GitHub
- Share OpenCPX with your network
- Write blog posts about your experience
- Present at meetups or conferences

### For Developers

#### Fix Bugs
Look for issues labeled `good first issue` or `help wanted`. These are great starting points for new contributors.

#### Add Features
Check the roadmap and open issues for features that interest you. Discuss your approach before starting large changes.

#### Improve SDKs
Contribute to the Go, Python, or JavaScript SDKs. Add features, fix bugs, or improve performance.

#### Write Tests
Increase test coverage, add edge cases, or improve test quality.

### For Technical Writers

#### Improve Documentation
- Fix typos and clarify confusing sections
- Add examples and tutorials
- Translate documentation

#### Create Guides
Write how-to guides for common use cases or specific frameworks.

### For Compliance Professionals

#### Review Schema Design
Provide feedback on whether the schema meets real-world compliance needs.

#### Contribute Framework Mappings
Add control mappings for compliance frameworks you're familiar with.

#### Share Use Cases
Document how OpenCPX solves problems in your organization.

## Getting Started

### Step 1: Explore the Project

- Read the [README](../README.md)
- Review the [specification](../spec/v1/spec.md)
- Try the [examples](../spec/v1/examples/)

### Step 2: Set Up Development Environment

```bash
git clone https://github.com/opencpx/OpenCPX.git
cd OpenCPX
```

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed setup instructions.

### Step 3: Find Your First Contribution

- Browse issues labeled [`good first issue`](https://github.com/opencpx/OpenCPX/labels/good%20first%20issue)
- Check [`help wanted`](https://github.com/opencpx/OpenCPX/labels/help%20wanted) issues
- Look at recent discussions for areas needing help

### Step 4: Make Your Contribution

1. Fork the repository
2. Create a branch
3. Make your changes
4. Submit a pull request

## Join Working Groups

Working groups are focused teams that drive specific aspects of OpenCPX.

### Schema Working Group
- **Focus**: Schema evolution and extensions
- **Good for**: Those interested in data modeling and standards
- **Meets**: Bi-weekly Tuesdays

### SDK Working Group
- **Focus**: SDK development across languages
- **Good for**: Developers who enjoy API design
- **Meets**: Bi-weekly Wednesdays

### Integrations Working Group
- **Focus**: Platform integrations
- **Good for**: Those familiar with compliance platforms
- **Meets**: Monthly, first Thursday

### Documentation Working Group
- **Focus**: Documentation and tutorials
- **Good for**: Writers and those who enjoy teaching
- **Meets**: Monthly, second Friday

See [meetings/](meetings/) for schedules and how to join.

## Communication Channels

### GitHub Discussions
For questions, ideas, and general discussion.

### GitHub Issues
For bug reports and feature requests.

### Working Group Meetings
For active collaboration on specific areas.

## Path to Maintainership

1. **Start contributing** - Code, docs, or community help
2. **Build expertise** - Focus on an area you're passionate about
3. **Demonstrate commitment** - Regular, quality contributions
4. **Help others** - Review PRs, answer questions
5. **Get nominated** - Committers can nominate active contributors

## Recognition

We recognize contributors in several ways:
- Listed in CONTRIBUTORS file
- Mentioned in release notes
- Featured in community spotlights
- Speaking opportunities at events
